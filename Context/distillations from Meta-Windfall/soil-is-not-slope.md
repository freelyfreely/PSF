The picker has no primary axis. Elevation correlates with temperature and rainfall but does not determine them—windward/leeward orientation, wind patterns, and microclimatic features (shady vs. cleared areas) create variations that no single parameter captures. The system reads the place as a whole: rainfall, temperature, soil type, elevation, light availability, and the user's own knowledge of their site's character.

This is not merely technical architecture but a stance toward land. Conventional zoning ("everything above 3000ft is upland rainy") imposes categories derived from abstract classification. The picker instead holds parameters lightly, as partial descriptions of a living system that exceeds them. The user's observation—"this part of my land is shady, that part is cleared"—becomes data equal in weight to GIS layers, because it describes the actual conditions plants will encounter.

The zone system for the flyers comes *after* the research, not before. Tam's initial categories were contradicted by the land itself (Waimea has dry areas above 3000ft). The field must be read from within, never assembled from without.
